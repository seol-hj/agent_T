# CI/CD 오류 수정 - 2026-05-11

## 문제 요약

### 1. ECR Repository Not Found
```
ERROR: failed to push 190484841865.dkr.ecr.ap-northeast-2.amazonaws.com/agent-t-prod/frontend:sha-fa23302
unknown: The repository with name 'agent-t-prod/frontend' does not exist
```

**원인**: 
- Workflow가 `agent-t-prod` ECR에 푸시 시도
- Terraform은 `agent-t-dev` ECR만 생성함

---

### 2. Dockerfile COPY Path Not Found
```
ERROR: failed to solve: failed to compute cache key: "/apps/analysis-service": not found
```

**원인**:
- Dockerfile: `COPY apps/analysis-service /app`
- Build context: `apps/analysis-service`
- → Dockerfile에서 `apps/` 경로를 찾을 수 없음

---

## 해결 방법

### 1. ECR Repository 이름 통일

#### 수정 전
```yaml
# .github/workflows/ci-frontend.yml
- name: Determine environment
  run: |
    if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
      echo "ecr-repo=agent-t-prod/frontend" >> $GITHUB_OUTPUT  # ❌ prod 없음
    else
      echo "ecr-repo=agent-t-dev/frontend" >> $GITHUB_OUTPUT
    fi
```

#### 수정 후
```yaml
# .github/workflows/ci-frontend.yml
- name: Determine environment
  run: |
    if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
      echo "ecr-repo=agent-t-dev/frontend" >> $GITHUB_OUTPUT  # ✅ dev 사용
    else
      echo "ecr-repo=agent-t-dev/frontend" >> $GITHUB_OUTPUT
    fi
```

**수정된 파일** (7개):
- `.github/workflows/ci-frontend.yml`
- `.github/workflows/ci-agent-service.yml`
- `.github/workflows/ci-analysis-service.yml`
- `.github/workflows/ci-api-service.yml`
- `.github/workflows/ci-report-service.yml`
- `.github/workflows/ci-simulation-runner.yml`
- `.github/workflows/ci-simulation-service.yml`

---

### 2. Build Context 수정

#### 문제 분석

**Dockerfile 구조** (Backend):
```dockerfile
# apps/analysis-service/Dockerfile
COPY libs /app/libs                      # 1. libs/ 복사 (루트)
COPY apps/analysis-service/requirements.txt .
COPY apps/analysis-service /app          # 2. 서비스 코드 복사
```

**기존 Build Context**:
```yaml
context-path: apps/analysis-service  # ❌ libs/ 접근 불가
dockerfile-path: apps/analysis-service/Dockerfile
```

Docker는 `apps/analysis-service`에서 빌드 시작 → `apps/` 경로 찾을 수 없음

---

#### 수정 전
```yaml
# .github/workflows/ci-analysis-service.yml
with:
  service-name: analysis-service
  dockerfile-path: apps/analysis-service/Dockerfile
  context-path: apps/analysis-service  # ❌
```

#### 수정 후
```yaml
# .github/workflows/ci-analysis-service.yml
with:
  service-name: analysis-service
  dockerfile-path: apps/analysis-service/Dockerfile
  context-path: .  # ✅ 루트 context (libs/ 접근 가능)
```

**수정된 서비스** (6개):
- `agent-service`
- `analysis-service`
- `api-service`
- `report-service`
- `simulation-runner`
- `simulation-service`

**Frontend는 변경 없음**:
- Frontend Dockerfile은 상대 경로만 사용
- `apps/frontend` context 유지

## Build Context 비교

### Backend Services

#### Before (❌ 오류)
```
Docker build context: apps/analysis-service/
├── Dockerfile
├── requirements.txt
└── app/
    ├── main.py
    └── ...

libs/  ← 접근 불가 (context 밖)
```

#### After (✅ 정상)
```
Docker build context: . (루트)
├── libs/              ← 접근 가능
│   └── common/
├── apps/
│   ├── analysis-service/
│   │   ├── Dockerfile
│   │   ├── requirements.txt
│   │   └── app/
│   └── ...
```

---

### Frontend

**변경 없음** (context 유지):
```
Docker build context: apps/frontend/
├── Dockerfile
├── package.json
├── src/
└── public/
```

**이유**: Frontend Dockerfile은 상대 경로만 사용
```dockerfile
COPY package.json ./
COPY . ./
# libs/ 불필요
```

- **Dev**: `agent-t-dev` ECR (존재 ✅)
- **Prod**: `agent-t-prod` ECR (없음 ❌)

### 향후 Production 배포 시

#### 1. Production ECR 생성
```bash
cd infra/terraform/envs/prod
terraform init
terraform apply
# → agent-t-prod ECR repositories 생성
```

#### 2. Workflow 수정
```yaml
# .github/workflows/ci-frontend.yml
- name: Determine environment
  run: |
    if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
      echo "ecr-repo=agent-t-prod/frontend" >> $GITHUB_OUTPUT  # ✅ prod 사용
```

**조건**:
- `main` 브랜치 → `agent-t-prod` (Production)
- `develop` 브랜치 → `agent-t-dev` (Dev)

## 대안: Branch 전략

### Option 1: 현재 방식 (단일 환경)
```yaml
# 모든 브랜치 → agent-t-dev
if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
  echo "ecr-repo=agent-t-dev/frontend"
else
  echo "ecr-repo=agent-t-dev/frontend"
fi
```

**장점**: 간단, Prod ECR 불필요  
**단점**: main/develop 구분 없음

---

### Option 2: 브랜치별 환경 (권장)
```yaml
# main → prod, develop → dev
if [[ "${{ github.ref }}" == "refs/heads/main" ]]; then
  echo "ecr-repo=agent-t-prod/frontend"
elif [[ "${{ github.ref }}" == "refs/heads/develop" ]]; then
  echo "ecr-repo=agent-t-dev/frontend"
else
  echo "ecr-repo=agent-t-dev/frontend"  # feature 브랜치
fi
```

**장점**: 환경 분리 명확  
**단점**: Prod ECR + Prod 인프라 필요

---

### Option 3: Tag 기반 배포
```yaml
# tag v* → prod, 나머지 → dev
if [[ "${{ github.ref }}" == refs/tags/v* ]]; then
  echo "ecr-repo=agent-t-prod/frontend"
else
  echo "ecr-repo=agent-t-dev/frontend"
fi
```

**장점**: Release 명확, main도 dev 사용 가능  
**단점**: Tag 관리 필요
