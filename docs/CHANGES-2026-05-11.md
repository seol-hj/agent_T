## 2. GitHub Actions Workflow 수정

### 문제
```
Invalid workflow file
(Line: 46, Col: 9): Unexpected value 'secrets'
```

**원인**: `workflow_call`을 사용할 때 `steps` 내에서 `uses`를 사용하면 안 됨

### 수정 파일 (7개)
- `.github/workflows/ci-frontend.yml`
- `.github/workflows/ci-agent-service.yml`
- `.github/workflows/ci-analysis-service.yml`
- `.github/workflows/ci-api-service.yml`
- `.github/workflows/ci-report-service.yml`
- `.github/workflows/ci-simulation-runner.yml`
- `.github/workflows/ci-simulation-service.yml`

### 변경 내용

#### Before (잘못된 문법)
```yaml
jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
      - name: Determine environment
        id: env
        run: |
          echo "ecr-repo=..." >> $GITHUB_OUTPUT

      - uses: ./.github/workflows/build-and-push.yml  # ❌ 잘못됨
        with:
          service-name: frontend
        secrets:  # ❌ steps 내에서 secrets 불가
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
```

#### After (올바른 문법)
```yaml
jobs:
  determine-env:
    runs-on: ubuntu-latest
    if: github.event_name == 'push'
    outputs:
      ecr-repo: ${{ steps.env.outputs.ecr-repo }}
    steps:
      - name: Determine environment
        id: env
        run: |
          echo "ecr-repo=..." >> $GITHUB_OUTPUT

  build-and-push:
    needs: determine-env
    uses: ./.github/workflows/build-and-push.yml  # ✅ job 레벨에서 사용
    if: github.event_name == 'push'
    with:
      service-name: frontend
      ecr-repository: ${{ needs.determine-env.outputs.ecr-repo }}
    secrets:  # ✅ job 레벨에서 secrets 전달
      aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
      aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
```

### 핵심 변경점

1. **Job 분리**
   - `determine-env` job: 환경 결정 (dev/prod)
   - `build-and-push` job: 재사용 가능한 workflow 호출

2. **`uses` 위치**
   - ❌ Before: `steps` 내에서 `uses`
   - ✅ After: `jobs.<job_id>` 레벨에서 `uses`

3. **Secrets 전달**
   - ❌ Before: `steps` 내에서 `secrets`
   - ✅ After: `jobs.<job_id>` 레벨에서 `secrets`

4. **Output 전달**
   - `outputs` 사용하여 job 간 데이터 전달
   - `needs.<job_id>.outputs.<key>` 형태로 참조

## 4. 참고 문서

### GitHub Actions workflow_call 문법
- 공식 문서: https://docs.github.com/en/actions/using-workflows/reusing-workflows

**올바른 사용법**:
```yaml
jobs:
  call-workflow:
    uses: ./.github/workflows/reusable.yml
    with:
      input1: value1
    secrets:
      secret1: ${{ secrets.SECRET1 }}
```

**잘못된 사용법**:
```yaml
jobs:
  my-job:
    runs-on: ubuntu-latest
    steps:
      - uses: ./.github/workflows/reusable.yml  # ❌ 오류
```
