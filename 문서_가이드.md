## 🚀 시나리오별 가이드

### 시나리오 1: "처음 프로젝트를 접했어요"

```bash
# 1. README 읽기
cat README.md

# 2. 로컬 실행
git clone <repo>
cd agent-t
docker compose up --build

# 3. 브라우저에서 테스트
open http://localhost:3000

# 4. 궁금하면 상세 문서 참조
```

**읽을 문서**:
1. README.md
2. QUICKSTART.md

---

### 시나리오 2: "코드를 수정하고 싶어요"

```bash
# 1. Git 전략 숙지
cat GITHUB_STRATEGY.md

# 2. 브랜치 생성
git checkout -b feature/my-feature

# 3. 아키텍처 이해
cat docs/architecture.md

# 4. 서비스 API 확인
cat docs/services.md

# 5. 개발 및 PR
...
```

**읽을 문서**:
1. GITHUB_STRATEGY.md
2. docs/architecture.md
3. docs/services.md
4. docs/contributing.md

---

### 시나리오 3: "AWS에 배포하고 싶어요"

```bash
# 1. 배포 가이드 정독
cat DEPLOYMENT.md

# 2. AWS 인증
aws configure

# 3. Terraform 실행
cd infra/terraform/envs/dev
terraform init
terraform apply

# 4. 서비스 배포
kubectl apply -f infra/argocd/applications/dev/
```

**읽을 문서**:
1. DEPLOYMENT.md
2. docs/infrastructure.md (Terraform 수정 시)
3. docs/cicd.md (자동화 설정 시)

---
